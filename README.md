# LIBRA World Manager v3.5.0

RisuAI용 장기 컨텍스트 관리 플러그인입니다.

LIBRA는 메모리, 엔티티, 관계, 세계관, 내러티브, 상태 추적, 세션/섹션 전환, 수동 로어북, 과거 대화 분석, 하이파 가져오기, 프롬프트 주입을 하나의 흐름으로 묶어 관리합니다.

## 핵심 기능

- 메모리 저장 / 검색 / GC / 중복 제거 / 턴 리커버리
- 엔티티 / 관계 추출, 병합, 수동 보호, 재분석
- 세계관 트리, 전역 세계 특성, 장면용 세계관 보정
- 내러티브 스토리라인 추적, 수동 보호, 재분석
- 스토리 작가 / 감독 프롬프트 계층
- 과거 대화 분석 / 과거 대화 재분석
- Hypa V3 지식 가져오기 및 누적 병합 + 검증
- 수동 로어북(`lmai_user_N`) 추가 및 구조 데이터 병합
- 일반 로어북 동적 참조(RAG)
- 감정 분석 / 장르 감지 / 자동 가중치 보정
- 작업 대시보드 및 모바일 대응 GUI
- CBS / GigaTrans / Lightboard 호환 옵션

## 데이터 구조

LIBRA는 `lmai_` 계열 로어북 엔트리를 사용합니다.

- `lmai_memory`: 일반 기억
- `lmai_entity`: 인물 / 엔티티 정보
- `lmai_relation`: 관계 정보
- `lmai_world_graph`: 세계관 그래프 / 전역 세계 특성
- `lmai_narrative`: 내러티브 스토리라인 상태
- `lmai_story_author`: 스토리 작가 상태
- `lmai_director`: 감독 상태
- `lmai_char_states`: 캐릭터 상태 추적
- `lmai_world_states`: 세계 상태 추적
- `lmai_user_*`: 유저 수동 로어북 원본

일반 로어북은 LIBRA 내부 구조 데이터와 분리되어 동적 참조(RAG)용 후보로 사용됩니다.

## 작동 방식 요약

### 1. 턴 처리

매 턴마다 LIBRA는 다음 흐름으로 동작합니다.

1. 현재 사용자 입력과 최근 상태를 기준으로 메모리 / 로어북 / 엔티티 / 관계 / 세계관 / 내러티브를 주입
2. 메인 응답 수신 후 메모리, 엔티티, 관계, 세계관 신호, 내러티브 턴 로그를 반영
3. 백그라운드에서 내러티브 요약, 캐릭터 상태, 세계 상태, 작가 / 감독 보조 상태를 후속 갱신
4. 필요하면 자동 상태 보정과 턴 리커버리로 누락된 턴을 회수

즉 모든 정보가 한 번에 갱신되는 것은 아니지만, 성격에 맞는 타이밍에 반영됩니다.

### 2. 프롬프트 주입

LIBRA는 `beforeRequest`에서 우선순위 기반으로 컨텍스트를 조립합니다.

- 최상위: 현재 유저 입력, 수동 로어북, 감독, 세계관, 지시사항
- 조건부: 엔티티, 관계, 장면용 세계관 보정, 캐릭터 상태
- 선택적: 메모리, 일반 로어북, 내러티브, 스토리 작가, 세계 상태

주입은 섹션별 압축과 우선순위 기반 예산 제어를 거치며, 중요 섹션은 유지하고 덜 중요한 섹션만 생략합니다.

### 3. 메모리와 구조 데이터

메모리는 중요도 / 감정 / 최신성 / 유사도를 기반으로 저장 / 검색됩니다.

구조 데이터는 별도로 저장됩니다.

- 엔티티 / 관계는 구조화된 JSON으로 관리
- 세계관은 계층형 노드 그래프로 관리
- 내러티브는 storyline 단위 상태로 관리
- 작가 / 감독 / 상태 추적기들은 별도 상태 엔트리로 유지

### 4. 세계관 시스템

세계관은 단순 텍스트가 아니라 구조 규칙으로 관리됩니다.

- `exists`: 마법, 기, 초자연, 기술, 종족, 신화적 존재
- `systems`: 레벨링, 스킬, 스탯, 클래스, 길드, 세력
- `physics`: 중력, 시간 흐름, 공간, 특수 현상
- `custom`: 세계별 추가 규칙

또한 전역 세계 특성을 따로 관리합니다.

- 멀티버스
- 차원 이동
- 시간 여행
- 메타 서술
- 가상현실
- 꿈 세계
- 회귀·환생·빙의
- 시스템 인터페이스

장면용 세계관 보정은 기본 세계관 프롬프트와 별도로, 현재 장면에서 어떤 세계관 요소를 우선 해석해야 하는지 메인 LLM이 보조 프롬프트로 생성하는 기능입니다.

### 5. 세션 / 섹션 전환

LIBRA는 채팅 / 캐릭터 / 페이지 / 섹션 단서를 묶은 `scopeKey` 기준으로 섹션 전환을 감지합니다.

- 다른 섹션으로 이동하면 섹션별 런타임 상태를 분리
- 같은 섹션으로 돌아오면 해당 섹션의 저장된 런타임 상태를 복원
- 전역 설정과 전역 세계관은 유지

### 6. 작업 대시보드

작업 중에는 대시보드가 표시되어 다음 정보를 보여줍니다.

- 주입된 컨텍스트 섹션
- 토큰 / 호출 상태
- 메인 LLM / 보조 LLM / 임베딩 호출 횟수
- 메인 작업 진행률
- 백그라운드 작업 진행률

작업 완료 후 약 5초 뒤 자동으로 닫힙니다.

## 과거 대화 분석 / 재분석

### 과거 대화 분석

과거 메시지를 읽어 초기 구조 데이터를 만듭니다.

흐름:

1. 분석 가능한 메시지 선별
2. 메시지를 청크로 분할
3. LLM이 청크 요약 생성
4. 합성 및 계층적 병합
5. 기존 구조 데이터와 검증 병합
6. 엔티티 / 관계 / 세계관 / 내러티브 / 전역 세계 특성 / 장면용 세계관 보정 반영

### 과거 대화 재분석

이미 존재하는 구조 데이터를 원문 대화 기준으로 다시 검토합니다.

현재 재분석 파이프라인:

1. 새 후보 구조 데이터 생성
2. 원문 대화 review window 단위 검토
3. review 결과와 기존 데이터 / 후보 데이터를 함께 판정
4. 최종 결과를 검증 병합
5. 엔티티 / 내러티브 / 세계관 GUI 즉시 갱신

현재는 `완전 교체`보다 `누적 병합 + 검증` 성향이 더 강합니다.

## Hypa V3 가져오기

Hypa V3 지식은 두 층으로 처리됩니다.

1. 원본을 `hypa_v3_import`와 `lmai_hypa_v3_source`로 보존
2. 구조화 분석 후 LIBRA 데이터에 누적 병합 + 검증

현재 반영 대상:

- 엔티티 / 관계
- 세계관 규칙
- 전역 세계 특성
- 내러티브 스토리라인
- 장면용 세계관 보정

같은 Hypa payload를 다시 가져오면 확인창이 뜹니다.

## 수동 로어북

GUI에서 `✍️ 수동 로어북 추가`를 사용하면 유저가 직접 손요약 / 설정 / 메모를 입력할 수 있습니다.

- 저장 키: `lmai_user_1`, `lmai_user_2` ...
- 저장 번호를 직접 지정 가능
- 같은 번호를 덮어쓸 때 확인창 표시
- 원본은 최상위 참고 로어북으로 저장
- 저장 직후 메인 LLM 분석을 거쳐 구조 데이터에 병합

수동 로어북은 일반 로어북보다 우선하는 상위 참고 자료로 주입됩니다.

## 수동 보호

엔티티 / 관계 / 내러티브는 GUI에서 저장하면 자동으로 수동 보호가 켜집니다.

- `수동 보호됨` / `자동 수정 가능` 배지 표시
- GUI에서 직접 보호 해제 / 재설정 가능
- 수동 보호된 필드는 자동 추출 / 자동 교정이 쉽게 덮어쓰지 않음

## 듀얼 LLM 구조

보조 LLM(`aux LLM`)을 활성화하면 다음 작업에 우선 사용됩니다.

- 과거 대화 분석 청크 요약
- 과거 대화 분석 합성
- 과거 대화 재분석 부분 review
- 유지관리 / 보조 판단 / 요약 작업

메인 LLM(`primary LLM`)은 다음에 사용됩니다.

- 실제 응답 생성
- 최종 병합 검증
- 장면용 세계관 보정 생성
- 수동 로어북 / Hypa / 분석 결과 구조화 반영의 핵심 판정

## 호환성 옵션

현재 설정에서 다룰 수 있는 호환 관련 기능:

- `CBS 엔진 사용`
- `로어북 동적 참조 (RAG)`
- `감정 분석 사용`
- `유저 무시 금지`
- `라이트보드 삽화 태그 호환`

기본 동작:

- GigaTrans 태그는 관리용 태그로 보고 필터링
- Lightboard 삽화 태그는 옵션을 켰을 때만 보존

## 디버그 로그

`설정 > 플러그인 기능 > 디버그 모드`를 켜면 콘솔에 다음이 출력됩니다.

- LLM 호출 시작 / 성공 / 실패
- 임베딩 호출 / 캐시 히트
- Memory retrieval 상세 점수
- Injected sections / 생략된 섹션
- Maintenance queue / background 상태
- Turn recovery / pending finalize 관련 로그
- 재분석 review / 최종 병합 흐름

## 권장 사용 흐름

1. 메인 LLM 설정
2. 필요하면 보조 LLM 설정
3. 필요하면 임베딩 설정
4. 메모리 프리셋 선택
5. 기존 채팅이면 과거 대화 분석 또는 재분석 실행
6. 필요하면 Hypa V3 가져오기 또는 수동 로어북 추가
7. 스토리 작가 / 감독 / 세계관 조정 모드 설정
8. 장기 서사는 섹션 전환과 세션 전환으로 이어가기

## 참고

- `main` 브랜치가 아니라 PR 브랜치에서 최신 기능이 먼저 반영될 수 있습니다.
- 일부 보조 기능은 보수적으로 동작하도록 설계되어, 근거가 약한 정보는 자동으로 채우지 않을 수 있습니다.
